## 已知条件

- ✓ 数据库集合已创建

## 可能原因及排查

### 1️⃣ 集合权限问题（最常见）

**检查步骤：**
1. 云开发控制台 → 数据库
2. 点击 `users` 集合
3. 点击"权限设置"
4. 查看当前权限配置

**测试期间建议配置：**
```json
{
  "read": true,
  "write": true
}
```

**或者使用自定义规则：**
```json
{
  "read": "auth.openid == doc.openid",
  "write": "auth.openid == doc.openid"
}
```

---

### 2️⃣ AppID 不匹配

**检查：**
1. project.config.json 中的 appid
2. 云开发环境所属的 AppID
3. 微信开发者工具登录的账号

**当前配置：**
```
appid: wx504a5b83dba5ce0b
```

如果云开发是用其他 AppID 创建的，需要：
- 切换到正确的 AppID
- 或重新创建云开发环境

---

### 3️⃣ 云开发未正确初始化

**在控制台执行测试：**
```javascript
// 测试 1
console.log('wx.cloud:', wx.cloud)

// 测试 2  
const db = wx.cloud.database()
console.log('database:', db)

// 测试 3
db.collection('users').limit(1).get()
  .then(res => console.log('✓ 查询成功:', res.data))
  .catch(err => console.error('✗ 查询失败:', err))
```

---

### 4️⃣ 查看具体错误信息

**在微信开发者工具中：**
1. 切换到"调试器"标签
2. 选择"Console"
3. 点击登录按钮
4. 查看红色错误信息

**常见错误码：**
- `-502001`: 集合不存在
- `-502002`: 权限不足
- `-1`: 网络错误
- `-501001`: 环境不存在

---

### 5️⃣ URL 校验域名问题

**检查 project.config.json：**
```json
"setting": {
  "urlCheck": false  // 测试期间设为 false
}
```

## 🔍 快速诊断脚本

在小程序控制台执行：

```javascript
const app = getApp()
console.log('=== 诊断信息 ===')
console.log('1. 云开发 SDK:', typeof wx.cloud)
console.log('2. 配置环境:', app.globalData.cloudEnv)
console.log('3. 数据库对象:', wx.cloud.database ? '存在' : '不存在')

wx.cloud.database().collection('users').limit(1).get()
  .then(res => {
    console.log('4. 数据库连接:', '✓ 成功')
    console.log('5. 数据条数:', res.data.length)
  })
  .catch(err => {
    console.error('4. 数据库连接:', '✗ 失败')
    console.error('错误码:', err.errCode)
    console.error('错误信息:', err.errMsg)
  })
```
